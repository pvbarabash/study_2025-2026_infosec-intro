# Цель работы

Получение практических навыков работы в консоли с атрибутами файлов для групп пользователей [@tuis].


# Выполнение лабораторной работы

**Задание 1.** В установленной операционной системе создайте учётную запись пользователя guest. Задайте пароль для пользователя guest. Аналогично создайте второго пользователя guest2.

Я получила права администратора командой su - и с помощью команды useradd guest (затем useradd guest2) создала пользователей guest и guest2, задала им пароли с помощью команды passwd (рис. [-@fig:001]).

![Создание пользователей под root](image/fig001.png){#fig:001 width=70%}


**Задание 2.** Добавьте пользователя guest2 в группу guest.

Я использовала команду gpasswd -a guest2 guest, чтобы добавить пользователя guest2 в группу guest (рис. [-@fig:002]).

![Добавление пользователя в группу](image/fig002.png){#fig:002 width=70%}

 
**Задание 3.** Осуществите вход в систему от двух пользователей на двух разных консолях: guest на первой консоли и guest2 на второй консоли

Я вошла в систему от имени пользователя guest с помощью команды su guest в одном терминале и повторила то же самое для guest2 в другом (рис. [-@fig:003]).

![Переход в учетные записи guest и guest2](image/fig003.png){#fig:003 width=70%}


**Задание 4.** Для обоих пользователей командой pwd определите директорию, в которой вы находитесь. Сравните её с приглашениями командной строки.

С помощью команды pwd я получила путь, он соответствует приглашению командной строки для обоих пользователей (рис. [-@fig:004]).

![Вывод пути местонахождения пользователей](image/fig004.png){#fig:004 width=70%}

**Задание 5.** Уточните имя вашего пользователя, его группу, кто входит в неё и к каким группам принадлежит он сам. Определите командами groups guest и groups guest2, в какие группы входят пользователи guest и guest2. Сравните вывод команды groups с выводом команд id -Gn и id -G. Сравните полученную информацию с содержимым файла /etc/group.

Я последовательно использовала команды id и groups для обоих пользователей, а также вывела информацию из файла /etc/group, используя команду cat, добавив выбор интересующей информации с помощью grep (рис. [-@fig:005]).

![Использование команд id и groups для пользователей, просмотр файла /etc/group](image/fig005.png){#fig:005 width=70%}

В выводе команды id указан идентификатор пользователя, указано имя пользователя, потом основная группа, к которой он принадлежит, а потом список всех групп, к которым он принадлежит.
В выводе команды groups указан список групп, к которым пользователь принадлежит.
Для пользователя guest указана только группа guest (в выводе обеих команды), а для пользователя guest2 указаны группы guest2 и guest (также для обеих команд).

В выдаче файла /etc/group можно увидеть, что в группу guest входит пользователь с идентификатором 1001 (по команде id мы знаем, что это пользователь guest), а также пользователь guest2. В группу guest2 входит только пользователь с идентификатором 1002, это пользователь guest2.


**Задание 6.** От имени пользователя guest2 выполните регистрацию пользователя guest2 в группе guest.

С помощью команды newgrp guest от имени пользователя guest2 я зарегистрировала пользователя guest2 в группе guest (рис. [-@fig:006]).

![Регистрация пользователя guest2 в группе guest](image/fig006.png){#fig:006 width=70%}

**Задание 7.** От имени пользователя guest измените права директории /home/guest, разрешив все действия для пользователей группы.

Командой chmod g+rwx /home/guest от имени пользователя guest я изменила права директории /home/guest, разрешив все действия для пользователей группы (рис. [-@fig:007]).

![Установка разрешений директории для пользователей групп](image/fig007.png){#fig:007 width=70%}

**Задание 8.** От имени пользователя guest снимите с директории /home/guest/dir1 все атрибуты и проверьте правильность снятия атрибутов.

Сначала я от имени пользователя guest создала директорию dir1 внутри директории /home/guest командой mkdir, затем командой chmod 000 /home/guest/dir1 также от того же guest сняла все атрибуты с директории и проверила правильность снятия командой ls -l (рис. [-@fig:008]).

![Снятие атрибутов на директорию и проверка правильности](image/fig008.png){#fig:008 width=70%}

Атрибуты были корректно сняты.

**Задание 9.** Меняя атрибуты у директории dir1 и файла file1 от имени пользователя guest и делая проверку от пользователя guest2, заполните таблицу, определив опытным путём, какие операции разрешены, а какие нет. Если операция разрешена, занесите в таблицу знак «+», если не разрешена, знак «-».


Я попеременно меняла права на директорию и файл от имени guest, а затем проверяла возможность действий от пользователя  guest2. Результат проверок можно увидеть в [табл. @tbl:01].

Для вместимости таблицы на странице мною использовались сокращения:

С. ф. — создание файла (командой touch);

У.ф. — удаление файла (командой rm);

Зап. в ф. — записать в файл (с помощью команды echo и перенаправления >);

Чт.ф — чтение файла (командой cat);

Смена дир. — смена директории (командой cd);

Просм. ф. в дир. — просмотр файла в директории (командой ls);

Переим. файл — переименование файла (командой mv dir1/file1 dir1/file3);

Смена атр. ф. — смена атрибутов файла (командой chmod).
